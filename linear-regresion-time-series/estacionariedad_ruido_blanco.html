Estacionariedad y Ruido Blanco

Propiedades estadísticas fundamentales para el modelaje de series de tiempo, con pruebas formales en Python

Datos: Series simuladas (y1 = ruido blanco, y2 = proceso autocorrelacionado) · Repositorio del curso

1. Estacionariedad (débil)

Una de las propiedades estadísticas más deseables en las series de tiempo es la estacionariedad. Intuitivamente, una serie es estacionaria cuando su comportamiento estadístico no cambia en el tiempo.

Formalmente, un proceso $\{y_t\}$ es débilmente estacionario (también llamado covarianza estacionario) si cumple tres condiciones:

Condición 1 — Media constante: $$E[y_t] = \mu < \infty$$ Condición 2 — Varianza constante (homoscedasticidad): $$\text{Var}[y_t] = \sigma^2 < \infty$$ Condición 3 — Autocovarianza depende solo de la distancia: $$\text{Cov}[y_t, y_{t-s}] = \gamma_s < \infty$$

La condición (1) implica que la serie fluctúa siempre alrededor de $\mu$, por lo tanto no puede tener tendencia. La condición (2) implica que la varianza sea la misma en todo momento — si hay "volatilidad agrupada" (periodos volátiles seguidos de periodos volátiles), la serie no es estacionaria. La condición (3) significa que la relación lineal entre observaciones solo depende de la distancia $s$ entre ellas, no del momento $t$ en que ocurren.

Estacionariedad estricta vs. débil

Un proceso es estrictamente estacionario si toda su distribución conjunta es invariante en el tiempo: $F\{y_1, \ldots, y_T\} = F\{y_{1+s}, \ldots, y_{T+s}\}$. Esto es mucho más restrictivo. En la práctica nos concentramos en la estacionariedad débil (las tres condiciones de arriba), que involucra solo los dos primeros momentos de la distribución (Alonso, 2020).

2. El ruido blanco

La piedra angular de todos los procesos que estudiaremos es el ruido blanco (white noise). Un proceso $\varepsilon_t$ es ruido blanco si:

Media cero: $$E[\varepsilon_t] = 0$$ Varianza constante: $$\text{Var}[\varepsilon_t] = \sigma_\varepsilon^2$$ No autocorrelación: $$\text{Cov}[\varepsilon_t, \varepsilon_s] = E[\varepsilon_t \varepsilon_s] = 0 \quad \text{para } t \neq s$$

La ausencia de autocorrelación implica que la serie no se puede predecir por medio de alguna combinación lineal de sus valores pasados. Si además asumimos normalidad, tenemos un ruido blanco gaussiano: $\varepsilon_t \sim \text{i.i.d.} \; N(0, \sigma_\varepsilon^2)$.

¿Por qué importa?

Un ruido blanco es, por definición, un proceso estacionario. Cuando modelamos series de tiempo (ARIMA, regresión, etc.), buscamos que los residuales del modelo se comporten como un ruido blanco. Si lo logran, significa que el modelo ha extraído toda la información predecible de la serie. Verificar esto equivale a comprobar los supuestos del Teorema de Gauss-Markov (Alonso, 2020, Cap. 4).

3. ¿Cómo determinar si una serie es ruido blanco?

Debemos verificar tres propiedades: no autocorrelación, homoscedasticidad, y (opcionalmente) normalidad. Usaremos dos series simuladas: y1 (un ruido blanco verdadero) y y2 (un proceso ARMA con autocorrelación).

4. Pruebas de autocorrelación

4.1 Prueba de Rachas (Wald-Wolfowitz, 1940)

Es una prueba no paramétrica que cuenta las "seguidillas" de signo (rachas) en la serie. Si no hay autocorrelación, los cambios de signo deberían ocurrir con una frecuencia esperada — ni demasiados (autocorrelación negativa) ni muy pocos (autocorrelación positiva).

Hipótesis: $$H_0: \rho = 0 \quad \text{(no hay autocorrelación)}$$ $$H_1: \rho \neq 0 \quad \text{(existe autocorrelación)}$$

El estadístico se basa en el número de rachas observadas $k$, con valor esperado y varianza:

$$E(k) = \frac{2 N_+ N_-}{N_+ + N_-} + 1 \qquad \text{Var}(k) = \frac{2N_+N_-(2N_+N_- - N_+ - N_-)}{(N_+ + N_-)^2(N_+ + N_- - 1)}$$
python
# ── Prueba de rachas para y1 (ruido blanco) ──
signo = data.y1 > 0
stat, pvalue = runstest_1samp(signo, correction=False)
print(f"y1 → Estadístico: {stat:.4f}, p-valor: {pvalue:.4f}")

# ── Prueba de rachas para y2 (autocorrelacionada) ──
signo2 = data.y2 > 0
stat2, pvalue2 = runstest_1samp(signo2, correction=False)
print(f"y2 → Estadístico: {stat2:.4f}, p-valor: {pvalue2:.2e}")
Salida y1 → Estadístico: -0.7446, p-valor: 0.4565 y2 → Estadístico: -6.5998, p-valor: 4.12e-11

Para y1, el p-valor = 0.4565 no permite rechazar $H_0$ (no hay evidencia de autocorrelación). Para y2, el p-valor ≈ 0 permite rechazar la hipótesis nula: la serie presenta autocorrelación significativa.

4.2 Prueba de Box-Pierce y Ljung-Box

Box y Pierce (1970) propusieron un test que verifica si las primeras $s$ autocorrelaciones son conjuntamente iguales a cero:

Estadístico de Box-Pierce: $$Q = n \sum_{j=1}^{s} r_j^2 \sim \chi^2_s$$ Corrección de Ljung-Box (para muestras pequeñas): $$Q' = n(n+2) \sum_{j=1}^{s} \frac{r_j^2}{n-j} \sim \chi^2_s$$

donde $r_j$ es la autocorrelación muestral al rezago $j$. La práctica común es probar para los primeros 20 rezagos:

python
# Prueba de Ljung-Box para y1 (primeros 20 rezagos)
Resul_lb_bp = sm.stats.acorr_ljungbox(
    data.y1, lags=range(1, 21),
    return_df=True, boxpierce=True
)

# Agregar columnas de decisión
Resul_lb_bp["lb_ho"] = np.where(
    Resul_lb_bp['lb_pvalue'] < 0.05,
    'rechazo_ho/hay_autocorr',
    'no_rechazo_ho/no_hay_autocorr'
)
Resul_lb_bp["bp_ho"] = np.where(
    Resul_lb_bp['bp_pvalue'] < 0.05,
    'rechazo_ho/hay_autocorr',
    'no_rechazo_ho/no_hay_autocorr'
)
print(Resul_lb_bp)

(rezagos 6-20: no rechazo en todos)

A pesar de que el primer rezago sugiere autocorrelación, para los demás 19 rezagos no se rechaza $H_0$. Se concluye que y1 se comporta como ruido blanco según esta prueba.

python
# Para y2: todas las pruebas rechazan H0
Resul_y2 = sm.stats.acorr_ljungbox(
    data.y2, lags=range(1, 21),
    return_df=True, boxpierce=True
)
print(Resul_y2[['lb_stat', 'lb_pvalue']].head())

Para y2 todos los p-valores son esencialmente cero: la serie tiene autocorrelación fuerte y no es ruido blanco.

5. Prueba de homoscedasticidad (efectos ARCH/GARCH)

Para detectar si la varianza tiene un comportamiento autocorrelacionado (efecto ARCH/GARCH — muy común en series financieras), aplicamos Ljung-Box sobre la serie al cuadrado centrada en su media: $(y_t - \bar{y})^2$.

python
# Ljung-Box sobre (y1 - media)^2 para detectar ARCH/GARCH
result_arch = sm.stats.acorr_ljungbox(
    (data.y1 - data.y1.mean())**2,
    lags=range(1, 21),
    return_df=True, boxpierce=True
)
print(result_arch[['lb_stat', 'lb_pvalue']].head(5))
Salida lb_stat lb_pvalue 1 0.572389 0.449311 2 0.915452 0.632721 3 6.285833 0.098502 4 6.405496 0.170843 5 7.434561 0.190275

Ningún rezago rechaza $H_0$: no hay comportamiento ARCH/GARCH en y1. La varianza es constante (homoscedasticidad).

6. Pruebas de normalidad

La normalidad no es necesaria para ser ruido blanco, pero sí para ser ruido blanco gaussiano (necesario para intervalos de confianza basados en la distribución normal).

6.1 Análisis gráfico: Q-Q Plot

python
# Gráfico Q-Q de normalidad
sm.qqplot(data.y1, line='45')
plt.title("Q-Q Plot — Serie y1")
plt.show()

# Histograma con curva normal teórica
mu, sigma = stats.norm.fit(data.y1)
x_hat = np.linspace(min(data.y1), max(data.y1), 100)
fig, ax = plt.subplots(figsize=(7, 4))
ax.hist(data.y1, density=True, bins=15, alpha=0.6, label="Histograma")
ax.plot(x_hat, stats.norm.pdf(x_hat, mu, sigma), 'r-', label=f"Normal(μ={mu:.2f}, σ={sigma:.2f})")
ax.legend(); plt.show()

6.2 Pruebas formales: Shapiro-Wilk y Jarque-Bera

python
# Prueba de Shapiro-Wilk
stat_sw, pval_sw = stats.shapiro(data.y1)
print(f"Shapiro-Wilk → Estadístico: {stat_sw:.4f}, p-valor: {pval_sw:.4f}")

# Prueba de Jarque-Bera
stat_jb, pval_jb = stats.jarque_bera(data.y1)
print(f"Jarque-Bera  → Estadístico: {stat_jb:.4f}, p-valor: {pval_jb:.4f}")
Salida Shapiro-Wilk → Estadístico: 0.9938, p-valor: 0.9310 Jarque-Bera → Estadístico: 0.3145, p-valor: 0.8545

Ambas pruebas no rechazan $H_0$ de normalidad. La serie y1 proviene de un proceso ruido blanco gaussiano.

Comentarios finales

Para determinar si una serie es estacionaria, el primer paso siempre es graficarla. Si tiene tendencia clara, probablemente no es estacionaria. Luego verificar las condiciones: primero la autocorrelación (lo más difícil de detectar "a ojo"), luego la homoscedasticidad, y finalmente la normalidad (opcional). En los próximos capítulos veremos pruebas formales de estacionariedad (como Dickey-Fuller Aumentada y KPSS) y cómo transformar series no estacionarias.
