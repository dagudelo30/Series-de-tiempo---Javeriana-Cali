Tendencia y Estacionalidad con Modelos de Regresión

Estimación de tendencia (lineal, cuadrática, polinómica) y estacionalidad determinística con variables dummy en Python

Datos: Tasa de desempleo mensual — 13 ciudades principales de Colombia (2001–2019) · Repositorio del curso

1. Contexto: Descomposición aditiva

Toda serie de tiempo puede descomponerse en sus componentes fundamentales. En la descomposición aditiva:

$$y_t = T_t + S_t + R_t$$

donde $T_t$ es el componente de tendencia (y ciclo), $S_t$ es el componente estacional y $R_t$ es el residuo aleatorio. En este capítulo estimamos $T_t$ y $S_t$ usando regresión por MCO (Mínimos Cuadrados Ordinarios), y verificamos que $R_t$ se comporte como un ruido blanco (Alonso, 2020, Cap. 5).

Marco teórico

La idea es sencilla: si la serie tiene tendencia, la modelamos como función del tiempo. Si tiene estacionalidad, la capturamos con variables dummy. Los pronósticos del modelo solo serán válidos si los residuales cumplen los supuestos del Teorema de Gauss-Markov (no autocorrelación, homoscedasticidad, normalidad). Por eso las pruebas del Capítulo 4 son indispensables aquí (Hyndman & Athanasopoulos, 2021, Cap. 7).

3. Tendencia lineal

El modelo más simple supone que la tendencia es una función lineal del tiempo:

$$y_t = \beta_1 + \beta_2 \cdot t + \varepsilon_t$$
python
# Crear variable de tiempo t = 1, 2, ..., T
x = np.linspace(1, data.shape[0], data.shape[0])
X = sm.add_constant(x)

# Ajustar regresión lineal por MCO
lin = sm.OLS(y, X)
lin_t = lin.fit()
print(lin_t.summary())
Resultados clave coef std err t P>|t| const 1.93e+01 0.281 68.740 0.000 x1 -3.53e-02 0.002 -20.318 0.000 R-squared: 0.652 Adj. R-squared: 0.651

El coeficiente de $t$ es significativo (p < 0.001), pero el $R^2$ = 0.652 indica que la tendencia lineal solo explica el 65% de la variación. Si graficamos los residuales, se observa que aún contienen tendencia no lineal y estacionalidad, lo cual viola los supuestos de Gauss-Markov.

python
# Visualizar tendencia y residuales
fig, axes = plt.subplots(1, 2, figsize=(14, 5))

axes[0].plot(y.values, label="Tasa desempleo")
axes[0].plot(lin_t.fittedvalues, label="Tendencia Lineal", color='red')
axes[0].legend(); axes[0].set_title("Serie + Tendencia lineal"); axes[0].grid()

axes[1].plot(lin_t.resid)
axes[1].set_title("Residuales"); axes[1].set_ylabel("Residuales"); axes[1].grid()
plt.tight_layout(); plt.show()
Residuales no aleatorios

Si los residuales muestran patrones (tendencia o estacionalidad), significa que el modelo aún tiene información por extraer. No debemos confiar en los pronósticos hasta que los residuales se comporten como ruido blanco.

4. Tendencia cuadrática

Si la tendencia no es lineal, podemos usar un polinomio de grado 2:

$$y_t = \beta_1 + \beta_2 \cdot t + \beta_3 \cdot t^2 + \varepsilon_t$$
python
# Agregar t² al modelo
data["x"] = np.linspace(1, data.shape[0], data.shape[0])
data["x2"] = data["x"]**2

X = sm.add_constant(data[["x", "x2"]])
pol = sm.OLS(y, X)
pol_2 = pol.fit()
print(pol_2.summary())
Resultados clave coef std err t P>|t| const 2.09e+01 0.256 81.619 0.000 x -7.61e-02 0.005 -14.756 0.000 x2 1.84e-04 0.000 8.206 0.000 R-squared: 0.803 Adj. R-squared: 0.801

El $R^2$ mejora a 0.803, pero los residuales aún muestran estacionalidad. Necesitamos incorporar el componente estacional.

python
# Pronóstico con tendencia cuadrática
h = 12
x_for = np.linspace(data.shape[0] + 1, data.shape[0] + h, h)
X_for = sm.add_constant(np.stack((x_for, x_for**2), axis=1))

dt = pol_2.get_prediction(X_for).summary_frame(alpha=0.05)
preds = dt[['mean', 'obs_ci_lower', 'obs_ci_upper']]
preds.columns = ['Point_forecast', 'lower_95', 'upper_95']
print(preds)

5. Estimación de la estacionalidad con variables dummy

La estacionalidad determinística aditiva se modela con variables dummy mensuales. Para datos mensuales se crean 11 dummies (una por cada mes excepto el de referencia) para evitar multicolinealidad perfecta:

$$y_t = \beta_1 + \sum_{i=2}^{12} \beta_i \cdot D_{i,t} + \varepsilon_t$$

donde $D_{i,t} = 1$ si $t$ corresponde al mes $i$. El intercepto $\beta_1$ captura el efecto del mes de referencia (enero), y cada $\beta_i$ representa la diferencia respecto a ese mes base.

python
# Crear dummies mensuales (drop_first=True para evitar multicolinealidad)
month_dummies = pd.get_dummies(
    data.index.month,
    drop_first=True,
    dtype=float
).set_index(data.index)

X = sm.add_constant(month_dummies)
sea = sm.OLS(y, X)
season = sea.fit()
print(season.summary())
Resultados clave R-squared: 0.134 Mes 2 (feb): coef = -0.73, p = 0.309 Mes 10 (oct): coef = -2.30, p = 0.002 (menor desempleo) Mes 1 (ene): intercepto = 15.08 (referencia) ...

El $R^2$ = 0.134 es bajo: la estacionalidad sola no explica bien la serie. Sin embargo, los residuales muestran que se capturó el patrón estacional, pero persiste la tendencia.

6. Modelo completo: Tendencia + Estacionalidad

Combinamos la tendencia polinómica con las variables dummy estacionales:

$$y_t = \beta_1 + \beta_2 \cdot t + \beta_3 \cdot t^2 + \sum_{i=2}^{12} \beta_{i+2} \cdot D_{i,t} + \varepsilon_t$$
python
# Tendencia cuadrática + estacionalidad
month_dummies = pd.get_dummies(
    data.index.month, drop_first=True, dtype=float
).set_index(data.index)

X = sm.add_constant(
    pd.concat([data[["x", "x2"]], month_dummies], axis=1)
)
season_pol2 = sm.OLS(y, X).fit()
print(season_pol2.summary())
Resultados clave R-squared: 0.916 Adj. R-squared: 0.911 F-statistic: 175.4 p-value: < 2.2e-16 Todos los coeficientes de tendencia y la mayoría de dummies estacionales son significativos (p < 0.05)

El $R^2$ salta a 0.916 — el modelo ahora explica más del 91% de la variación. Pero la pregunta clave es: ¿los residuales cumplen los supuestos?

python
# Verificar residuales
plt.figure(figsize=(12, 4))
plt.plot(season_pol2.resid)
plt.title("Residuales: Tendencia cuadrática + Estacionalidad")
plt.ylabel("Residuales"); plt.grid(); plt.show()

7. Evaluación de los supuestos de los residuales

Para que los pronósticos e intervalos de confianza sean válidos, los residuales deben comportarse como ruido blanco (ver Capítulo 4). Debemos verificar:

7.1 No autocorrelación (Ljung-Box)

python
result_lb = sm.stats.acorr_ljungbox(
    season_pol2.resid, lags=range(1, 21),
    return_df=True, boxpierce=True
)
print(result_lb[['lb_stat', 'lb_pvalue']])

7.2 No heteroscedasticidad (ARCH/GARCH)

python
# Ljung-Box sobre residuales al cuadrado
result_arch = sm.stats.acorr_ljungbox(
    season_pol2.resid**2,
    lags=range(1, 21),
    return_df=True
)
print(result_arch[['lb_stat', 'lb_pvalue']].head())

7.3 Normalidad (Jarque-Bera y Shapiro-Wilk)

python
from scipy import stats

# Jarque-Bera
jb_stat, jb_pval = stats.jarque_bera(season_pol2.resid)
print(f"Jarque-Bera: stat={jb_stat:.4f}, p={jb_pval:.4f}")

# Shapiro-Wilk
sw_stat, sw_pval = stats.shapiro(season_pol2.resid)
print(f"Shapiro-Wilk: stat={sw_stat:.4f}, p={sw_pval:.4f}")

8. Comparación de modelos fuera de muestra

La comparación definitiva siempre se hace fuera de muestra. Siguiendo el enfoque del libro, podemos evaluar diferentes órdenes polinómicos combinados con estacionalidad:

python
# Comparación de modelos con diferentes órdenes polinómicos
results = {}
for name, model in [
    ("Lineal", lin_t),
    ("Cuadrática", pol_2),
    ("Solo Estacional", season),
    ("Cuadrática + Estacional", season_pol2)
]:
    results[name] = {
        'R²': model.rsquared,
        'R² adj': model.rsquared_adj,
        'AIC': model.aic,
        'BIC': model.bic
    }

comparison = pd.DataFrame(results).T
print(comparison.round(4))

9. Limitaciones y consideraciones

Sobreajuste con polinomios altos: A medida que aumenta el orden del polinomio, el ajuste in-sample mejora pero los pronósticos fuera de muestra pueden volverse muy inestables. Polinomios de orden 7+ tienden a "explotar" en la extrapolación (Alonso, 2020).

Estacionalidad determinística: El modelo asume que el patrón estacional es fijo y predeterminado. Si la estacionalidad cambia en el tiempo, se necesitan modelos más sofisticados (estacionalidad estocástica, SARIMA).

Validación de supuestos: Los pronósticos e intervalos de confianza solo son válidos si los residuales cumplen los supuestos de Gauss-Markov. Siempre verificar autocorrelación, homoscedasticidad y (para intervalos paramétricos) normalidad.

Multicolinealidad: Al usar polinomios, las variables $t$, $t^2$, $t^3$ están correlacionadas. Aunque esto no sesga los coeficientes, puede inflar los errores estándar. Considerar usar polinomios ortogonales si esto es un problema.
